# 阻止 Unity 自动导入示例的解决方案

## 问题分析
Unity 包管理器会自动导入包中的示例文件到 `Assets/Samples/` 文件夹。即使删除，下次导入时也会重新创建。

## 解决方案

### 方案 1: 在 Package Manager 中禁用示例导入
1. 打开 **Package Manager** (`Window > Package Manager`)
2. 找到 **XR Interaction Toolkit**
3. 点击右侧的 **Samples** 选项卡
4. 不要点击 **Import** 按钮
5. 或者导入后立即删除

### 方案 2: 修改 manifest.json 添加排除设置
```json
{
  "dependencies": {
    "com.unity.xr.interaction.toolkit": "3.1.2",
    // ... 其他包
  },
  "scopedRegistries": [],
  "disableSampleImport": true  // 可能不支持此选项
}
```

### 方案 3: 创建 .sampleignore 文件（如果支持）
```bash
# 在项目根目录创建
echo "XR Interaction Toolkit" > .sampleignore
```

### 方案 4: 使用脚本自动清理
创建编辑器脚本，在每次导入后自动删除 Samples 文件夹。

## 立即执行的操作

### 步骤 1: 彻底清理
```bash
# 删除所有 Samples 相关文件
rm -rf Assets/Samples*
rm -f Assets/Samples*.meta

# 清理可能残留的文件
find . -name "*Sample*" -type d | grep -v ".git" | xargs rm -rf 2>/dev/null
```

### 步骤 2: 重新导入包（不导入示例）
1. 在 Package Manager 中
2. 选择 XR Interaction Toolkit
3. **不要点击** Samples 选项卡中的 Import
4. 或者导入后立即在编辑器中删除

### 步骤 3: 验证清理效果
1. 重新编译项目
2. 检查 Console 错误数量
3. 确认 Samples 文件夹不再出现

## 预防措施

### 创建自动清理脚本
```csharp
// Assets/Editor/AutoCleanSamples.cs
using UnityEditor;
using UnityEngine;
using System.IO;

[InitializeOnLoad]
public class AutoCleanSamples
{
    static AutoCleanSamples()
    {
        EditorApplication.delayCall += CleanSamplesFolder;
    }
    
    static void CleanSamplesFolder()
    {
        string samplesPath = "Assets/Samples";
        if (Directory.Exists(samplesPath))
        {
            Directory.Delete(samplesPath, true);
            File.Delete(samplesPath + ".meta");
            AssetDatabase.Refresh();
            Debug.Log("已自动清理 Samples 文件夹");
        }
    }
}
```

### 修改项目设置
在 `ProjectSettings/ProjectSettings.asset` 中添加：
```yaml
EditorSettings:
  m_EnableSampleImport: 0
```

## 验证步骤

1. 删除现有 Samples 文件夹
2. 重新打开 Unity 项目
3. 等待包重新导入
4. 检查 Samples 是否重新出现
5. 检查编译错误数量

## 预期结果

- Samples 文件夹不再自动创建
- 61个红色错误减少到 0-5个
- 项目可以正常编译
